# Codex Frontend QA

Build UI that is useful, grounded in the product, and verified in a browser.

## Workflow

1. Inspect existing design system, components, routes, data contracts, and target
   audience before designing.
2. Build the actual tool or workflow first, not a marketing shell, unless the
   request is explicitly for a landing page.
3. Use real assets, domain-specific visuals, or generated bitmap assets when a
   visual experience is expected.
4. Keep layouts responsive with stable dimensions for boards, toolbars, cards,
   grids, counters, and controls.
5. Avoid text overflow, overlapping UI, nested cards, decorative noise, and
   one-note palettes.
6. Verify rendered behavior with browser checks, screenshots, responsive
   viewports, and interaction tests where relevant.

## Frontend Claims

- Code inspection is not enough for layout quality.
- A build passing does not prove the UI is usable.
- If browser tooling fails, record the limitation separately from product state.

## Output

- Summarize changed surfaces, browser/viewports checked, screenshots or routes
  inspected, and unresolved UI risks.
